## 네트워크 관련 신기술

### 소프트웨어 정의 기술

- 네트워크, 데이터 센터 등에서 소유한 자원을 가상화해서 갭려 사용자에게 제어하고 중앙에서 관리하는 기술
- `SDN(Software Defined Networking)`: `네트워크`를 가상화하고 소프트웨어로 제어
- `SDDC(Software Defined Data Center)`: `데이터 센터`의 모든 자원을 가상화하고 소프트웨어로 제어
- `SDS(Software Defined Storage)`: `스토리지 자원`을 가상화하고 소프트웨어로 관리

### 기타 용어

- `올조인(AllJoyn)`: 다양한 제조사의 기기들이 상호 운용될 수 있도록 지원하는 오픈 소스 프레임워크
- `메시 네트워크`: 대규모 무선 네트워크
- `피코넷`: 여러 블루투스 기술이나 UWB 통신 기술을 쓰는 장치가 무선 망을 형성
- `Zing`: 저전력, 저비용 무선 통신 기술(NFC)
- `WDM`: 광섬유 통신에서 여러 파장의 빛을 동시에 전송

## 네트워크 구축

- `CAMS/CD`: `데이터 프레임` 충돌 감지, 재전송 기능
- `CSMA/CA`: `무선 랜`에서 충돌 회피

## 경로 제어

- IGP(Interior Gateway Protocol)
    - RIP(Routing Information Protocol):
        - 벨만 포드 알고리즘 기반 라우팅 프로토콜
        - 최대 홉 수 15로 제한(소규모 전용)
    - OSPF(Open Shortest Path First)
        - 다익스트라 알고리즘 기반 라우팅 프로토콜
        - 대규모 네트워크에 적합

## 트래픽 제어

- 정지-대기: 수신 측의 확인 신호를 받은 후에 패킷을 전송하는 방식

## SW 신기술

- `Vaporware`: 발표되었지만 아직 배포되지 않은 소프트웨어

## 보안 관련 신기술

- `비트로커`: windows 전용 볼륨 암호화 기능
- `OWASP`: 보안 관련 취약점 연구하는 비영리 단체
- `TCP wrapper`: 외부 컴퓨터의 접근을 제어하는 리눅스 보안 프로그램
- `허니팟`: 해커의 공격을 유인하여 분석하는 보안 시스템
- `DPI(Deep Packet Inspection)`: OSI 7 계층의 전 층 포로토콜과 패킷 내부를 검사

## 회복

- `즉각 갱신 기법`: Redo, Undo 모두 가능

## 병행 제어

- `로킹(locking)`
    - 로킹 단위가 작으면 관리하기 어려우나 병행도가 높아짐
- `타임 스탬프 순서`

## 교착 상태

### 필요 충분 조건

- `상호 배제`: 자원을 한 프로세스가 독점
- `점유와 대기`: 프로세스가 점유된 자원을 기다림
- `비선점`: 자원을 강제로 빼앗을 수 없음
- `환형 대기`: 프로세스들이 원형으로 자원을 기다림

### 해결 방법

- `회피 기법`: 은행원 알고리즘
